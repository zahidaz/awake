<h2>Where to Start</h2>

<div class="awake-roles">
  <div class="awake-role">
    <strong>Malware Analyst</strong>
    <p>Start with the <a href="malware/families/">family catalog</a>. The <a href="malware/timeline/">timeline</a> gives historical context, and <a href="malware/naming-conventions/">naming conventions</a> helps map between vendor detection names. When a sample is packed, use the <a href="packers/#analysis-decision-tree">packer decision tree</a>.</p>
  </div>
  <div class="awake-role">
    <strong>Reverse Engineer</strong>
    <p>Start with <a href="packers/">packer identification</a>. For runtime work, <a href="reversing/hooking/">hooking</a> and <a href="reversing/patching/">patching</a> cover the major approaches. The <a href="reversing/frameworks/">frameworks section</a> documents each development framework's analysis workflow.</p>
  </div>
  <div class="awake-role">
    <strong>Pentester</strong>
    <p>Start with <a href="attacks/">attack techniques</a> -- organized by attack surface with a combination matrix. The <a href="permissions/">permissions section</a> documents what each permission unlocks and how escalation works in practice.</p>
  </div>
  <div class="awake-role">
    <strong>Threat Intelligence</strong>
    <p>Start with <a href="malware/threat-actors/">threat actors</a> for MaaS operator attribution. The <a href="malware/timeline/">timeline</a> tracks evolution from 2010 to present. <a href="grayware/">Grayware</a> covers the ecosystem between monetization and malware.</p>
  </div>
</div>
